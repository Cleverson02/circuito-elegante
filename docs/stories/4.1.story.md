## Descricao

Este story implementa a **camada de recepcao de mensagens WhatsApp** via Evolution API — a fundacao de todo o canal WhatsApp da Stella. E o primeiro story do EPIC-4 e habilita todos os stories subsequentes (4.2 buffer, 4.3 typing, 4.4 renderizacao, etc.).

**Por que Evolution API e nao Cloud API direta (ADR-5):**

A decisao arquitetural ADR-5 escolheu Evolution API por ser open-source, suportar nativamente composing/recording events, read receipts controlados, presenca online dinamica, e ter integracao nativa com Chatwoot — tudo essencial para a experiencia de luxo da Stella. A Evolution API roda como container Docker self-hosted (`http://evolution-api:8080/`), dando controle total sobre latencia e privacidade dos dados.

**O que este story entrega:**

1. **Evolution API Client** — Modulo TypeScript que encapsula toda comunicacao com a Evolution API (send text, send media, update presence, mark as read, composing events). Segue o mesmo padrao do ElevareClient existente em `backend/src/integrations/elevare/`.
2. **Webhook Receiver** — Rota Fastify `POST /webhook/whatsapp` que recebe eventos da Evolution API (`messages.upsert`, `messages.update`, `connection.update`), valida assinatura, faz parse do payload e enfileira para processamento.
3. **Read Receipts Inteligentes (FR34)** — Marca mensagens como "lidas" apenas APOS o buffer de 20s e no inicio do processamento, nunca no recebimento. Evita o "vacuo de luxo" (mensagem vista sem resposta).
4. **Presenca Online Dinamica (FR35)** — Stella nao fica "Online" 24/7. Estado "Online" ativado apenas durante processamento ativo. Fora disso: offline (parece humana).
5. **Log Estruturado** — Cada mensagem recebida gera log com correlationId, source, eventType, phoneNumber, sessionId, channel, mediaType.

**Decisao critica: Webhook handler retorna 200 OK imediatamente.** Todo processamento e async para evitar timeouts da Evolution API. A mensagem e enfileirada para o buffer de 20s (Story 4.2) que consolidara mensagens fragmentadas antes de enviar ao pipeline de agentes.

**Nota sobre o deliverMessage TODO:** O `server.ts` atual (Story 3.7) tem um TODO para integrar com Evolution API no Epic 4. Este story resolve esse TODO conectando o webhook follow-up delivery ao Evolution API Client.
