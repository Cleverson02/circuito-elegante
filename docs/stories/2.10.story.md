## Descrição

Implementar processamento multimodal nativo para Stella: áudio (mensagens de voz do WhatsApp), imagens (fotos de hotéis, comprovantes, screenshots) e inputs mistos (texto + mídia). O Input Handler é o gateway que detecta o tipo de mídia, processa para formato textual enriquecido e alimenta o pipeline existente (Intent → Orchestrator → Persona → Safety).

**Problema/Necessidade:** 40-60% das interações via WhatsApp no mercado brasileiro envolvem áudio. Sem processamento multimodal, Stella ignora a maioria das mensagens de voz dos guests e não consegue interpretar fotos enviadas (comprovantes de pagamento, fotos de hotéis para identificação, screenshots de cotações).

**Decisão Arquitetural:** Processar mídia no Input Handler ANTES do pipeline de agentes. Áudio é transcrito para texto + contexto emocional. Imagens são descritas em tokens visuais. O pipeline downstream recebe sempre texto enriquecido — zero mudança nos agentes existentes.

## Dev Notes

### Arquitetura Multimodal

```
WhatsApp (Evolution API)
    ↓
Input Handler (detect type)
    ├── text → pass through
    ├── audio → AudioProcessor → Whisper → texto + emoção
    ├── image → ImageProcessor → Vision → descrição + OCR
    └── mixed → combina resultados
    ↓
[Texto enriquecido]
    ↓
Pipeline existente (Intent → Orchestrator → Persona → Safety)
```

### Formatos WhatsApp
- **Áudio:** OGG/Opus (padrão WhatsApp), também MP3, WAV, M4A
- **Imagem:** JPEG, PNG (WhatsApp comprime para JPEG), WebP
- **Documento:** PDF (comprovantes, vouchers)

### OpenAI APIs utilizadas
- **Whisper:** `openai.audio.transcriptions.create({ model: 'whisper-1', file })` — max 25MB
- **Vision:** `openai.chat.completions.create({ messages: [{ content: [{ type: 'image_url', image_url }] }] })` — modelo GPT-4o nativo
- **NSFW:** Usar Moderation API `openai.moderations.create({ input: imageDescription })` como segunda camada

### Performance Targets
- **Áudio (30s):** < 3s processamento total (transcrição + enrichment)
- **Imagem:** < 2s processamento total (vision + OCR + NSFW check)
- **Mixed:** < 5s processamento total
- **NFR12:** Pipeline completo com multimodal < 10s p95

### Contexto Emocional do Áudio
Não usar análise de espectrograma — extrair emoção do TEXTO transcrito:
- Palavras-chave: "urgente", "preciso", "por favor", exclamações repetidas → `urgent`
- Tom frustrado: "já pedi", "ninguém responde", negações → `frustrated`
- Entusiasmo: "incrível", "adorei", superlativos → `excited`
- Neutro: ausência dos padrões acima → `neutral`

### Segurança
- **NUNCA** armazenar mídia original permanentemente — processar e descartar
- **NUNCA** logar conteúdo de áudio transcrito em plain text — apenas metadata
- Imagens inapropriadas: rejeitar ANTES de enviar para Vision API (pré-check leve)
- Rate limit específico para mídia: max 10 files/min por sessão
