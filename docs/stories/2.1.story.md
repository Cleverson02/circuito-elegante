## Descrição

Implementar o LLM Client agnóstico com provider OpenAI (GPT-5 family) e o Intent Agent usando GPT-5-nano para classificação de intenções. O Intent Agent é o primeiro estágio do pipeline multi-agente — recebe a mensagem do usuário e classifica em uma das 7 intenções definidas, detectando idioma e suportando multi-intenção.

**Problema/Necessidade:** O pipeline de agentes precisa de uma camada de abstração LLM e de um classificador de intenções como primeiro estágio. Sem o Intent Agent, não há como rotear mensagens para os tools corretos.

**Decisão Arquitetural:** Usar OpenAI Agents SDK (JS/TS) nativo — ADR-1 validado via OpenAI CS Agents Demo. Handoffs nativos + guardrails + tracing integrados.

## Dev Notes

- **Pipeline Architecture:** Intent → Orchestrator → Persona → Safety (4 agentes sequenciais)
- **Model Assignments:** Intent=nano, Orchestrator=turbo, Persona=pro, Safety=nano
- **Guardrails executam em paralelo** antes do pipeline, zero latência percebida
- **OpenAI Agents SDK:** `new Agent({ model, instructions, tools, inputGuardrails })` + `Runner.run(agent, input)`
- **Referência:** OpenAI CS Agents Demo (airline→hotel adaptation)
- Consultar `docs/architecture/architecture.md` seção 5 para detalhes do pipeline
