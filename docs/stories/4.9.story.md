## Descricao

Este story implementa o **sistema de WhatsApp Template Messages** ‚Äî mensagens pro-ativas pre-aprovadas pelo Meta que Stella pode enviar fora da janela de 24h do WhatsApp Business API.

**Problema/Necessidade:** O WhatsApp Business API tem uma regra fundamental: apos 24h sem interacao do cliente, a empresa so pode enviar **template messages** pre-aprovadas pelo Meta. Sem templates, Stella nao pode:
- Notificar que uma cotacao esta expirando (FR25)
- Confirmar uma reserva (FR25)
- Fazer follow-up apos atendimento fora de expediente (Story 4.7)
- Reengajar um hospede que nao respondeu

**4 Templates Necessarios:**

| Template | Evento | Variaveis |
|----------|--------|-----------|
| `follow_up_quote_expiring` | Cotacao proxima de expirar | `guest_name`, `hotel_name`, `expiry_time` |
| `reservation_confirmed` | Reserva confirmada | `guest_name`, `hotel_name`, `check_in_date`, `check_out_date` |
| `out_of_hours_followup` | Mensagem recebida fora do expediente | `guest_name` |
| `reengagement` | Pos-atendimento sem conversao | `guest_name`, `hotel_name` |

**Processo Meta:** Templates devem ser submetidos ao Meta para aprovacao (fora do escopo deste story ‚Äî documentacao preparada para submissao). Enquanto nao aprovados, o sistema faz graceful fallback (log warning + skip).

**Integracao com Evolution API:** A Evolution API expoe endpoint `POST /message/sendTemplate/{instance}` que aceita template name + variables. Este story cria o renderer de templates e integra com o client Evolution API.

### Data Models
- **TemplateDefinition:** `{ name: string, language: string, variables: string[], body: string }`
- **RenderedTemplate:** `{ name: string, language: string, body: string, parameters: Array<{ type: 'text', text: string }> }`
- **TemplateSendResult:** `{ sent: true, response: EvolutionResponse } | { sent: false, reason: string }`

### Evolution API Template Payload
```json
{
  "number": "5511999999999",
  "template": {
    "name": "follow_up_quote_expiring",
    "language": { "code": "pt_BR" },
    "components": [{
      "type": "body",
      "parameters": [
        { "type": "text", "text": "Maria" },
        { "type": "text", "text": "Hotel Fasano Rio" },
        { "type": "text", "text": "18:00 de hoje" }
      ]
    }]
  }
}
```

### Technical Constraints
- **Template names devem corresponder EXATAMENTE** ao nome registrado no Meta ‚Äî qualquer divergencia retorna erro.
- **Language code** deve ser `pt_BR` (nao `pt` ou `pt-BR`) ‚Äî formato especifico do WhatsApp Business API.
- **Variaveis sao posicionais** no Meta ‚Äî a ordem dos parameters deve corresponder a ordem das `{{1}}`, `{{2}}`, etc. no template aprovado. O body local usa nomes (`{{guest_name}}`), mas o payload para a API usa indices.
