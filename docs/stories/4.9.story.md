## Descricao

Este story implementa o sistema de **WhatsApp Template Messages** — mensagens pre-aprovadas pelo Meta que permitem Stella enviar mensagens **proativas** fora da janela de 24h do WhatsApp. Sem templates aprovados, Stella so pode responder dentro de 24h apos a ultima mensagem do hospede.

**Por que templates sao criticos:**

O WhatsApp Business API impoe uma regra: apos 24h sem interacao do hospede, a empresa so pode enviar **template messages** pre-aprovados pelo Meta. Sem isso, cenarios como "cotacao expirando", "confirmacao de reserva", e "follow-up fora de expediente" simplesmente nao funcionam.

**Templates definidos (4):**
1. `follow_up_quote_expiring` — "Ola {{guest_name}}, seu link de reserva para {{hotel_name}} expira em {{expiry_time}}..."
2. `reservation_confirmed` — "{{guest_name}}, sua reserva no {{hotel_name}} esta confirmada! Check-in: {{check_in}}..."
3. `out_of_hours_followup` — "{{guest_name}}, recebemos sua mensagem! {{message_summary}}..."
4. `reengagement` — "{{guest_name}}, encontramos opcoes incriveis no {{hotel_name}}..."

**Nota Sprint 0:** Story adicionada pela validacao Aria/Pax. A implementacao de codigo e completa, mas os templates reais precisam ser submetidos e aprovados pelo Meta Business Manager (processo manual fora do escopo de codigo).

## Acceptance Criteria

- [ ] AC1: Templates definidos em `backend/src/integrations/templates/definitions.ts` com schema Zod para cada template: nome, variaveis obrigatorias, texto base (PT), categoria Meta (UTILITY ou MARKETING)
- [ ] AC2: Funcao `renderTemplate(templateName: string, variables: Record<string, string>): string` que substitui `{{variable}}` pelos valores fornecidos. Throw se variavel obrigatoria ausente
- [ ] AC3: Metodo `sendTemplate(phone, templateName, variables)` adicionado ao `EvolutionClient` — POST a `/message/sendTemplate/{instance}` com `{ number, template, params }`

- [ ] AC6: Testes: (a) renderTemplate substitui variaveis corretamente, (b) variavel ausente → throw, (c) sendTemplate com mock, (d) fallback quando nao aprovado
- [ ] AC7: Barrel export em `backend/src/integrations/templates/index.ts`

## Tasks / Subtasks

- [ ] **T1: Template Definitions** (AC1)
  - [ ] Criar `backend/src/integrations/templates/definitions.ts`
  - [ ] Schema Zod para cada template:
    ```typescript
    { name: string, category: 'UTILITY' | 'MARKETING', requiredVars: string[], textPT: string }
    ```
  - [ ] Definir 4 templates:
    - `follow_up_quote_expiring`: vars `[guest_name, hotel_name, expiry_time]`, category UTILITY
    - `reservation_confirmed`: vars `[guest_name, confirmation_number, check_in, check_out]`, category UTILITY
    - `out_of_hours_followup`: vars `[guest_name, message_summary]`, category UTILITY
    - `reengagement`: vars `[guest_name, hotel_name, recommendation]`, category MARKETING

- [ ] **T2: Template Renderer** (AC2, AC7)
  - [ ] Criar `backend/src/integrations/templates/renderer.ts`
  - [ ] `renderTemplate(templateName, variables)`:
    - Lookup template por nome em definitions
    - Validar variaveis obrigatorias presentes → throw `MissingTemplateVariableError` se ausente
    - Substituir `{{var}}` pelo valor
    - Retornar texto renderizado
  - [ ] Criar `backend/src/integrations/templates/index.ts` barrel export

- [ ] **T3: EvolutionClient.sendTemplate** (AC3, AC4)
  - [ ] Adicionar metodo ao `backend/src/integrations/evolution/client.ts`:
    ```typescript
    async sendTemplate(phone: string, templateName: string, variables: Record<string, string>): Promise<{ sent: boolean; reason?: string }>
    ```
  - [ ] POST a `/message/sendTemplate/{instance}` com `{ number: phone, template: templateName, params: variables }`
  - [ ] Se sucesso → `{ sent: true }`
  - [ ] Se erro 400/403 (template nao aprovado) → log warning, retornar `{ sent: false, reason: 'not_approved' }`
  - [ ] Se outro erro → log error, rethrow

- [ ] **T4: Documentacao Playbook** (AC5)
  - [ ] Criar `docs/playbooks/whatsapp-templates.md` com:
    - Lista de todos 4 templates
    - Para cada: nome, categoria Meta, variaveis com descricao, texto completo PT
    - Instrucoes passo-a-passo para submissao ao Meta Business Manager
    - Checklist de aprovacao (guidelines Meta para templates)

- [ ] **T5: Testes Unitarios** (AC6)
  - [ ] Criar `tests/unit/whatsapp-templates.test.ts`:
    - Teste: renderTemplate substitui todas variaveis corretamente
    - Teste: variavel obrigatoria ausente → throw MissingTemplateVariableError
    - Teste: template inexistente → throw TemplateNotFoundError
    - Teste: sendTemplate com mock → chamada correta a Evolution API
    - Teste: sendTemplate com erro 400 → `{ sent: false, reason: 'not_approved' }` (nao throw)
    - Teste: cada template renderizado com variaveis de exemplo → output correto
