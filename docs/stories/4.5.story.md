## Descricao

Este story conecta o mecanismo de **correfencia** (Story 3.5) ao **WhatsApp Reply nativo**. Quando o hospede faz Reply a uma imagem especifica de suite enviada pela Story 4.4, Stella resolve automaticamente qual oferta o hospede esta referenciando ‚Äî sem precisar perguntar "qual opcao voce quer?".

**Fluxo:**
1. Story 4.4 envia 3-4 imagens de suites ‚Üí Evolution API retorna `messageId` para cada
2. Story 4.4 chama `updateOfferMapMessageIds(sessionId, { position: messageId })` para popular o mapa
3. Hospede faz Reply a imagem 2 ‚Üí webhook recebe `quotedMessageId` (do `ParsedMessage`)
4. Story 4.5 chama `resolveCoreference(sessionId, { whatsappMessageId: quotedMessageId })`
5. `resolveByMessageId()` encontra posicao 2 ‚Üí retorna oferta completa (offerId, roomId, ratePlanId)
6. Pipeline processa a escolha sem expor IDs tecnicos

**O que ja existe (Story 3.5):** O modulo `coreference.ts` ja implementa `resolveByMessageId()`, `updateOfferMapMessageIds()`, e `MessageIdNotFoundError`. Este story apenas **conecta** o webhook handler ao resolver via `quotedMessageId`.

- [ ] AC1: Quando webhook recebe mensagem com `isReply: true` e `quotedMessageId` preenchido, extrair o `quotedMessageId` do `ParsedMessage` e passar para o pipeline de correfencia
- [ ] AC2: No pipeline, chamar `resolveCoreference(sessionId, { whatsappMessageId: quotedMessageId })` que internamente usa `resolveByMessageId()` existente
- [ ] AC3: Se resolucao bem-sucedida ‚Üí retornar oferta resolvida (offerId) para o Orchestrator Agent prosseguir com quotation
- [ ] AC4: Se `quotedMessageId` nao encontrado no OfferMap ‚Üí `MessageIdNotFoundError` capturado gracefully ‚Üí tratar como mensagem normal (texto da reply processado pelo pipeline textual)
- [ ] AC5: Story 4.4 deve chamar `updateOfferMapMessageIds(sessionId, positionToMessageIdMap)` apos enviar cada imagem ‚Äî **verificar que esta integracao existe** (ou implementar se ausente)
- [ ] AC6: Teste: enviar reply a imagem na posicao 2 ‚Üí resolver para offerId correto da posicao 2
- [ ] AC7: Teste: reply a messageId nao mapeado ‚Üí fallback para resolucao textual (sem erro)
- [ ] AC8: Log estruturado: `{ event: 'coreference_whatsapp_reply', sessionId, quotedMessageId, resolvedOfferId, resolvedPosition }`
