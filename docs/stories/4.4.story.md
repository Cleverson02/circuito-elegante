## Descricao

Este story implementa a **renderizacao premium de opcoes de hotel** no WhatsApp — o momento visual da experiencia Stella. Em vez de listar opcoes como texto puro (comportamento de chatbot generico), Stella envia cada opcao como **imagem nativa do WhatsApp** com legenda formatada contendo nome da suite, preco em BRL e amenidades em bullets.

**Por que renderizacao premium e essencial:**

Hospedes de luxo sao visuais. Uma foto da suite com vista para o mar vale mais que mil palavras. A renderizacao premium transforma a curadoria de 3-4 opcoes (Story 3.4) em uma experiencia visual imersiva que maximiza conversao. O hospede ve a suite, le o preco formatado, e responde "quero essa" — sem precisar imaginar com base em descricao textual.

**Golden Discovery #1:** As fotos vem diretamente na resposta `/search` da Elevare (`ElevarePhoto.url`). Sao assets transacionais — nao precisam de storage proprio. Story 4.4 simplesmente passa a URL para `EvolutionClient.sendMedia()`.

**Integracao com Story 4.3 (Typing Simulation):**

O typing queue (Story 4.3) tem o flag `isMedia: true` no `TypingChunk`. Quando `isMedia: true`, o worker envia a imagem **instantaneamente** (sem typing event, sem delay). A legenda subsequente segue a cadencia normal de digitacao humana. Isso cria o efeito: foto aparece imediato → "digitando..." → legenda aparece com ritmo natural.

```
Opcao 1: [FOTO suite] → instantaneo
         [Legenda formatada] → com typing simulation
Opcao 2: [FOTO suite] → instantaneo
         [Legenda formatada] → com typing simulation
...
```

## Acceptance Criteria

- [ ] AC1: Servico `renderCuratedOptions(options: CuratedOption[], phone: string, sessionId: string, channel: 'whatsapp' | 'website'): TypingChunk[]` em `backend/src/services/media-renderer.ts` que transforma opcoes curadas em chunks de imagem + legenda
- [ ] AC2: Para cada `CuratedOption`, gerar 2 chunks: (1) chunk de imagem com `{ text: photoUrl, isMedia: true, delay: 0 }` e (2) chunk de legenda com `{ text: captionFormatada, isMedia: false, delay: calculateDelay(caption) }`
- [ ] AC3: Foto extraida de `option.photos[0]` (primeira foto do array). Se `photos` vazio ou `photos[0].url` ausente → fallback (AC6)
- [ ] AC4: Legenda formatada conforme template: `{roomType}\n\nR$ {displayPricePerNight}/noite\nR$ {displayPrice} total ({nights} noites)\n\n{amenities em bullets}`. Usar `formatPriceBRL()` de `curadoria.ts` para formatacao de precos
- [ ] AC5: Amenidades renderizadas como bullets (max 4): `• {amenity1}\n• {amenity2}\n• {amenity3}`. Se nenhuma amenidade → omitir secao
- [ ] AC6: Fallback quando imagem indisponivel — em vez de chunk de imagem, gerar chunk de texto: `"🏨 {roomType} — {displayPrice} total (foto indisponivel)"`. Log warning `media_photo_unavailable` com offerId e photoUrl
- [ ] AC7: Funcao `formatCaption(option: CuratedOption): string` exportada em `backend/src/services/caption-formatter.ts` que gera o texto da legenda
- [ ] AC8: Suporte multi-idioma na legenda — labels ("por noite", "total", "noites") baseados no idioma da sessao (PT default, EN e ES como futuro). Neste story: apenas PT hardcoded, com estrutura preparada para i18n
- [ ] AC9: Validacao de URL de imagem antes de criar chunk — verificar que URL inicia com `http://` ou `https://` e nao esta vazia. URL invalida → fallback (AC6)
- [ ] AC10: Integracao com pipeline — no ponto onde Persona Agent retorna resposta com opcoes curadas, chamar `renderCuratedOptions()` para gerar chunks visuais em vez de texto puro. Manter texto puro para `channel: 'website'`
- [ ] AC11: Para canal `website`, gerar chunks de texto simples (sem imagem nativa) — imagens serao renderizadas pelo chat widget (Story 4.6)
- [ ] AC12: Log estruturado para cada opcao renderizada: `{ event: 'option_rendered', sessionId, offerId, hasPhoto, photoUrl, captionLength, channel }`
- [ ] AC13: Testes unitarios cobrindo: (a) formatacao de legenda com precos BRL, (b) geracao de chunks imagem + legenda, (c) fallback sem foto, (d) URL invalida, (e) amenidades vazias, (f) canal website vs whatsapp — cobertura >= 90% branches
- [ ] AC14: Funcao `validatePhotoUrl(url: string): boolean` exportada para reuso futuro

### Technical Constraints
- **Evolution API aceita URL direta** — nao precisa fazer download local da imagem. Passa a URL e a Evolution API faz o fetch [Source: architecture.md#section-6.2]
- **Max 4 amenidades em bullets** — para nao sobrecarregar a legenda no WhatsApp [Source: PRD FR17]
- **WhatsApp markup** — suporta *bold*, _italic_, ~strikethrough~. Usar `*{roomType}*` para nome da suite [Source: WhatsApp formatting]
- **Primeiro chunk delay 0** — primeira imagem deve chegar instantaneamente (NFR7). O `isMedia: true` ja garante isso no worker [Source: 4.3.story.md#AC9]
- **Fallback NAO deve quebrar o fluxo** — se imagem indisponivel, enviar texto e continuar. Log warning para monitoring [Source: PRD FR22]
