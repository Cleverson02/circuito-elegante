## Descricao

Este story implementa o **chat widget basico para website** — o segundo canal de comunicacao da Stella alem do WhatsApp. E um widget embed HTML/JS que conecta via WebSocket ao backend Fastify, permitindo hospedes interagir com Stella diretamente no site do hotel.

**Diferenca critica vs WhatsApp:**
- **Sem buffer 20s** — website = resposta imediata (nao tem mensagens "picadas")
- **Typing simulation via WebSocket** — blocos de texto enviados com delays (reusa Story 4.3)
- **Sem media nativa** — imagens renderizadas como `<img>` tags no widget (nao sendMedia)

**Arquitetura:**
```
Hospede abre site do hotel
    ↓
Widget carrega (HTML/JS embed)
    ↓
Conecta WebSocket ws://backend:3000/ws/chat
    ↓
Envia mensagem texto
    ↓
Backend processa via pipeline (Intent → Orchestrator → Persona → Safety)
    ↓
Typing queue envia chunks via WebSocket (channel: 'website')
    ↓
Widget renderiza mensagens com animacao de typing
```

## Acceptance Criteria

- [ ] AC1: Dependencia `ws` adicionada ao `package.json`
- [ ] AC2: WebSocket server em `backend/src/api/websocket.ts` criado e integrado ao Fastify via `app.server` (upgrade handler)
- [ ] AC3: Rota WebSocket `ws://host:port/ws/chat` aceita conexoes e gera `sessionId` unico por conexao
- [ ] AC4: Cliente envia `{ type: 'message', text: string }` → backend processa via pipeline existente com `channel: 'website'`
- [ ] AC5: Backend responde via WebSocket com `{ type: 'message', text: string, timestamp: number }` para cada chunk do typing queue
- [ ] AC6: Typing indicator enviado como `{ type: 'typing', isTyping: boolean }` antes de cada chunk (simula "digitando...")
- [ ] AC7: Sessao WebSocket mapeada em Redis com `channel: 'website'` — reusa `SessionData` existente
- [ ] AC8: Sem buffer 20s — mensagem processada imediatamente ao receber (sem MessageBuffer)
- [ ] AC9: Widget HTML/JS embed em `frontend/widget/` — arquivo unico `stella-widget.js` que cria iframe ou div com chat UI
- [ ] AC10: Widget responsivo — funciona em desktop (largura fixa 380px) e mobile (full width)
- [ ] AC11: Cor primaria verde CE `#289548` aplicada no header, botao enviar, e balao de mensagem da Stella
- [ ] AC12: Widget mostra historico da sessao atual (mensagens persistem durante a conexao WebSocket)
- [ ] AC13: Graceful disconnect — se WebSocket desconecta, widget mostra mensagem "Reconectando..." e tenta reconnect automatico (max 3 tentativas, backoff 2s/4s/8s)
- [ ] AC14: Testes unitarios do WebSocket handler: conexao, envio de mensagem, typing indicator, sessao, disconnect
- [ ] AC15: Server.ts atualizado para inicializar WebSocket server no bootstrap

## Dev Notes

### Technical Stack
- **WebSocket library:** `ws` (lightweight, nativo Node.js) — NAO socket.io [Source: architecture.md — lightweight preference]
- **Fastify integration:** `ws` attacha no `app.server` (raw http.Server) via upgrade event [Source: ws docs]
- **Redis sessao:** Reusa `SessionData` com `channel: 'website'` de `backend/src/state/session-manager.ts` [Source: session-manager.ts]
