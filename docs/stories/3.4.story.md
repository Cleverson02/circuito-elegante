## Descricao

Implementar o **Motor de Curadoria** — o algoritmo inteligente que transforma uma lista crua de 10-50 opcoes retornadas pelo Elevare `/search` em uma selecao refinada de 3-4 opcoes representativas para o hospede. Este e o componente que materializa o paradigma de UX "Curadoria Ativa (Nao Dumping)" definido no PRD (secao 3.3): Stella filtra opcoes complexas em um recorte representativo, como um concierge de luxo faria presencialmente.

**Por que isso e critico para a experiencia AAA:**

Um concierge de hotel cinco estrelas nunca entregaria 50 panfletos ao hospede. Ele ouviria as necessidades, analisaria internamente todas as opcoes e apresentaria 3-4 sugestoes cuidadosamente selecionadas, garantindo diversidade de preco (uma opcao acessivel, uma intermediaria, uma premium/exclusiva) e destacando o que torna cada uma unica. Se o hospede quisesse ver mais, o concierge apresentaria o proximo grupo. A curadoria e o que separa Stella de um buscador generico.

**Paradigma tecnico:** O motor de curadoria e uma **funcao pura** (pure function) — recebe dados de entrada (array de ofertas Elevare + configuracao de hotel) e retorna dados de saida (array curado de 3-4 opcoes + metadados de paginacao). Zero side effects, zero I/O, zero dependencia de estado externo. Isso garante testabilidade total e determinismo.

**Fluxo no pipeline:**
```
Elevare /search (Story 3.1)
  -> Array de ElevareOffer[] (10-50 opcoes)
  -> Motor de Curadoria (ESTA STORY)
    -> input: ElevareOffer[] + { bradescoCoupon: boolean }
    -> output: CuratedResult { options: CuratedOption[3-4], hasMore: boolean, page: number }
  -> Persona Agent formata para prosa AAA (Story 2.5)
  -> Estado Oculto / Correfencia mapeia offerIds (Story 3.5)
```

**Bradesco coupon:** Para hoteis que aceitam cupom Bradesco (`hotels.bradescoCoupon = true`), o preco exibido ao hospede inclui desconto de 10% (`totalPrice * 0.90`). O preco original e preservado integralmente no `originalPrice` para uso na cotacao (Story 3.3) — o desconto e APENAS visual, aplicado na camada de apresentacao.

| Risco | Probabilidade | Impacto | Mitigacao |
|-------|---------------|---------|-----------|
| Algoritmo de buckets produz selecao nao-representativa quando distribuicao de precos e extremamente skewed (ex: 45 opcoes baratas + 5 caras) | Media | Medio | Buckets sao por cardinalidade (nao por range de preco), entao 33% de cada faixa por count garante pelo menos 1 opcao de cada tier. Testes cobrem distribuicao skewed |
| Bradesco coupon calculation com floating point errors (ex: `R$ 1.000 * 0.90 = R$ 899.9999...`) | Media | Alto — preco incorreto e inaceitavel em AAA | `Math.round(value * 100) / 100` garante arredondamento centesimal. ## Dev Notes

### Algoritmo de Curadoria — Pseudocodigo Completo

```
FUNCTION curateOptions(offers: ElevareOffer[], config: CuradoriaConfig): CuratedResult

  IF offers.length === 0:
    RETURN empty result (options=[], hasMore=false, page=1)

  // Step 1: Sort deterministically
  sorted = offers.sort((a, b) => {
    if (a.totalPrice !== b.totalPrice) return a.totalPrice - b.totalPrice;
    return a.offerId.localeCompare(b.offerId);  // deterministic tiebreak
  })

  IF sorted.length <= 3:
    RETURN all offers as CuratedOptions (no bucketing needed)

  // Step 2: Split into 3 buckets by cardinality
  bucketSize = floor(sorted.length / 3)
  budgetBucket  = sorted[0 .. bucketSize-1]
  midBucket     = sorted[bucketSize .. 2*bucketSize-1]
  premiumBucket = sorted[2*bucketSize .. end]

  // Step 3: Select best from each bucket
  selected = [
    selectBest(budgetBucket,  'budget'),
    selectBest(midBucket,     'mid-range'),
    selectBest(premiumBucket, 'premium'),
  ]

  // Step 4: Add wildcard if enough options
  IF sorted.length >= 10:
    wildcard = selectWildcard(sorted, selected.map(s => s.offerId))
    IF wildcard !== null:
      selected.push(wildcard)

  // Step 5: Apply Bradesco discount
  IF config.bradescoCoupon:
    selected = selected.map(opt => applyBradescoDiscount(opt, true))

  // Step 6: Assign ranks and metadata
  selected.forEach((opt, i) => opt.curatedRank = i + 1)

  RETURN {
    options: selected,
    hasMore: offers.length > selected.length,
    page: 1,
    totalAvailable: offers.length,
    priceRange: { min: sorted[0].totalPrice, max: sorted[sorted.length-1].totalPrice },
    priceVariety: calculatePriceVariety(sorted),
  }


FUNCTION selectBest(bucket: ElevareOffer[], label: PriceBucket): CuratedOption
  // Priority: photos > amenities > lowest price
  withPhotos = bucket.filter(o => o.photos.length > 0)
  candidates = withPhotos.length > 0 ? withPhotos : bucket
  candidates.sort((a, b) => {
    if (a.amenities?.length !== b.amenities?.length)
      return (b.amenities?.length || 0) - (a.amenities?.length || 0);
    return a.totalPrice - b.totalPrice;
  })
  RETURN toCuratedOption(candidates[0], label)


FUNCTION selectWildcard(sorted: ElevareOffer[], alreadySelectedIds: string[]): CuratedOption | null
  remaining = sorted.filter(o => !alreadySelectedIds.includes(o.offerId))
  IF remaining.length === 0: RETURN null
  remaining.sort((a, b) => (a.totalPrice / a.nights) - (b.totalPrice / b.nights))
  RETURN toCuratedOption(remaining[0], 'wildcard')
```

### Bradesco Coupon Logic — Detalhes

O campo `hotels.bradescoCoupon` (boolean) indica se o hotel aceita cupom Bradesco com 10% de desconto. A informacao vem do PostgreSQL (tabela `hotels`, Story 1.2) e e passada ao motor de curadoria via `CuradoriaConfig.bradescoCoupon`.

```typescript
// Aplicacao do desconto
const applyBradescoDiscount = (option: CuratedOption, enabled: boolean): CuratedOption => {
  if (!enabled) {
    return { ...option, displayPrice: option.totalPrice, displayPricePerNight: option.pricePerNight, hasBradescoDiscount: false };
  }
  const displayPrice = Math.round(option.totalPrice * 0.90 * 100) / 100;
  const displayPricePerNight = Math.round(displayPrice / option.nights * 100) / 100;
  return {
    ...option,
    displayPrice,
    displayPricePerNight,
    hasBradescoDiscount: true,
    originalPrice: option.totalPrice,  // SEMPRE preservado
  };
};

// IMPORTANTE: originalPrice e o que vai para POST /quotations (Story 3.3)
// displayPrice e o que o hospede ve na mensagem WhatsApp
// Nunca usar displayPrice para cotacao — apenas para apresentacao
```

### Paginacao "Ver Mais Opcoes" — Session Flow

A paginacao e stateless no motor de curadoria (funcao pura). O Orchestrator gerencia o estado de pagina na sessao Redis:

```
Hospede: "Quero ver quartos disponiveis para agosto"
  -> Elevare /search retorna 25 opcoes
  -> curateOptions(offers, config) -> pagina 1 (3-4 opcoes curadas por price bucket)
  -> Persona Agent apresenta opcoes

Hospede: "Tem mais opcoes?"
  -> curateNextPage(offers, config, page=2) -> pagina 2 (proximas 4 opcoes por preco)
  -> Persona Agent apresenta opcoes

Hospede: "Mais alguma coisa?"
  -> curateNextPage(offers, config, page=3) -> pagina 3 (proximas 4 opcoes)
  -> Se hasMore=false, Persona diz: "Estas sao todas as opcoes disponiveis para o periodo."
```

**Redis keys para paginacao** (gerenciados pelo Orchestrator, NAO por este modulo):
```
curadoria:{sessionId}:offers   -> ElevareOffer[] completo (ref: request:{requestId})
curadoria:{sessionId}:page     -> numero da pagina atual (default 1)
curadoria:{sessionId}:config   -> CuradoriaConfig { bradescoCoupon, maxOptions }
```
