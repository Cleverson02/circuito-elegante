## Descrição

Criar middleware de sanitização de input que remove caracteres de controle, detecta injection attempts e limita tamanho de mensagem. Atualizar o rate limiting existente (Story 1.5) para os limites do pipeline de agentes (30 msgs/min, 500 msgs/dia).

**Problema/Necessidade:** Input do usuário precisa ser sanitizado antes de entrar no pipeline de agentes. Injection attacks podem manipular prompts ou extrair dados sensíveis.

---

## Acceptance Criteria

- [ ] AC1: Sanitize middleware: remove control chars, detecta injection, max 2000 chars
- [ ] AC2: Se injection detectada: log Sentry + resposta persona intacta
- [ ] AC3: Rate limiting: 30 msgs/min, 500 msgs/dia (atualizar Story 1.5)
- [ ] AC4: Se excedido: resposta educada em vez de erro técnico
- [ ] AC5: Parâmetros tools sanitizados (Drizzle parameterized)
- [ ] AC6: Testes para cada vetor de ataque
