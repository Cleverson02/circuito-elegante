## Descrição

Este story implementa o passo FINAL do Golden Discovery #2 — o "Fluxo Mágico" de reservas. Após a busca de disponibilidade (Story 3.1, que produz `requestId`) e o registro do hóspede (Story 3.2, que produz `customerId`), resta apenas combinar estes dois IDs com o `offerId` selecionado pelo hóspede para gerar um link de pagamento via `POST /quotations`.

**Problema/Necessidade:** Stella precisa converter a intenção de reserva em um link de pagamento funcional sem expor IDs técnicos ao hóspede. O link tem validade limitada (`expiresAt`), exigindo monitoramento de expiração, regeneração automática (FR29) e extensão de validade (FR30) — tudo transparente para o usuário final.

**Decisão Arquitetural:** O módulo `quotations.ts` é isolado do `client.ts` base (separação por domínio, mesmo padrão de `customers.ts`). O estado da cotação é mantido em Redis com chaves dedicadas (`quotation:{id}`) para rastreamento de ciclo de vida do link. O `requestId` NUNCA transita pelo WhatsApp — é recuperado do Redis no momento da criação da cotação (FR24).

**Fluxo Completo:**
```
requestId (Redis, Story 3.1) + offerId (seleção do hóspede) + customerId (Story 3.2)
  → POST /quotations
  → quotationId + paymentLink + expiresAt
  → Redis quotation:{id} (rastrear expiração)
  → Hóspede recebe apenas o link humanizado
```

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|---------------|---------|-----------|
| requestId expira (TTL 30min) antes do hóspede confirmar reserva | Alta | Alto — precisa refazer search | Detectar expiração early, sugerir nova busca com dados da sessão. Considerar estender TTL do requestId para 45min em Story 3.1 se frequente |
| Payment link expira sem o hóspede clicar | Média | Médio — precisa regenerar | `regeneratePaymentLink()` (FR29) evita refazer todo o fluxo. Webhook `quote_expiring` (Story 3.7) notifica proativamente |
| offerId inválido (hóspede demora e opção sai de disponibilidade) | Média | Alto — API retorna 4xx | Capturar erro específico, informar hóspede com linguagem elegante, sugerir nova busca |
| Race condition: dois requests de quotation simultâneos para mesmo requestId | Baixa | Médio — duplicidade | Serializar por sessionId (Redis lock ou check-before-create) |
| Mismatch entre auth headers (x-client-id vs X-Api-Key) | Baixa | Alto — 401 em todas as calls | Confirmar com Postman collection (headers documentados: `x-client-id` + `x-client-secret`) |

## Dev Notes

### Elevare Quotation API — Contract Details

**POST /global-agent/quotations** (Create Quotation)
```json
// Request
{
  "customerId": "cust_abc123",
  "offerId": 0,                           // integer! NOT string
  "requestId": "req_xyz789",
  "validityDays": 1,                      // default 1 day
  "includePaymentLink": true,             // always true for our flow
  "customerMessage": "Segue sua cotação conforme solicitado...",
  "internalNotes": "Cliente solicitou quarto com vista para o mar"
}

// Response (expected)
{
  "data": {
    "id": "quot_abc123",                  // quotationId
    "paymentLink": "https://pay.elevare.com.br/...",
    "expiresAt": "2026-04-06T14:30:00Z",
    "status": "active"
  }
}
```

**PUT /global-agent/quotations/{id}/payment-link** (Regenerate — FR29)
```json
// Request
{
  "validityDays": 2,
  "reason": "Cliente solicitou extensão do prazo"
}

// Response (expected)
{
  "data": {
    "paymentLink": "https://pay.elevare.com.br/...",
    "expiresAt": "2026-04-08T14:30:00Z"
  }
}
```

**PUT /global-agent/quotations/{id}/extend** (Extend Validity — FR30)
```json
// Request
{
  "additionalDays": 1,
  "generateNewPaymentLink": true
}

// Response (expected)
{
  "data": {
    "expiresAt": "2026-04-07T14:30:00Z",
    "paymentLink": "https://pay.elevare.com.br/..."  // only if generateNewPaymentLink=true
  }
}
```

### Redis Key Patterns for Quotation State

```
quotation:{quotationId}     → QuotationState JSON     (TTL: expiresAt + 1h)
session_quotations:{sid}    → Set of quotationIds      (TTL: session TTL = 24h)
request:{requestId}         → RequestData JSON         (TTL: 30min, from Story 3.1)
```

**QuotationState** stored in Redis:
```json
{
  "quotationId": "quot_abc123",
  "paymentLink": "https://pay.elevare.com.br/...",
  "expiresAt": "2026-04-06T14:30:00Z",
  "status": "active",
  "sessionId": "sess_xyz",
  "requestId": "req_xyz789",
  "offerId": 42,
  "customerId": "cust_abc123",
  "createdAt": "2026-04-05T14:30:00Z",
  "updatedAt": "2026-04-05T14:30:00Z"
}
```

### requestId Flow (FR24: State Isolation)

```
[Story 3.1] GET /search → requestId → Redis request:{requestId} (TTL 30min)
[Story 3.3] Redis request:{requestId} → validate exists → POST /quotations body
                                                            ↑ NEVER from WhatsApp
```

The `requestId` is an opaque internal token. The hóspede never sees it. The Orchestrator retrieves it from Redis using the session context. If it has expired, the system must gracefully suggest a new search rather than exposing a technical error.

### Quotation State Machine

```
created ──(5min)──→ active ──(expiresAt - 2h)──→ expiring ──(expiresAt)──→ expired
                                                      │                        │
                                                      ├─ regenerate() ────→ active
                                                      └─ extend() ────────→ active
```

- `created`: Transitional state immediately after API response (within 5 min of creation)
- `active`: Payment link is valid and has > 2h remaining
- `expiring`: Payment link will expire within 2h — trigger proactive notification (Story 3.7)
- `expired`: Payment link is no longer valid — offer regeneration (FR29) or new search

### Error Scenarios and Recovery

| Scenario | Error Type | Recovery |
|----------|-----------|----------|
| requestId expired in Redis | `RequestExpiredError` | Suggest new search, preserve session context |
| offerId not a valid integer | `InvalidOfferError` | Re-prompt for selection |
| offerId no longer available | `QuotationApiError` (from 4xx) | Inform hóspede, suggest alternatives |
| Elevare API 5xx | `QuotationApiError` | Retry via base client, then fallback (Story 3.8) |
| Elevare API timeout (8s) | `QuotationApiError` | Retry, then fallback |
| quotationId not found (regenerate) | `QuotationNotFoundError` | Create new quotation |
| Payment link already used | `QuotationApiError` | Inform hóspede payment was processed |
| Network error | `QuotationApiError` | Retry via base client circuit breaker |
