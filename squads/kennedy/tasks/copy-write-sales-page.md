## Purpose

Write a web sales page adapting Kennedy's direct mail principles to the digital environment. The fundamental rules of direct response copy don't change — but the format must adapt to how people consume web content.

> "A web sales page is just a sales letter on a screen. The psychology is identical. The only difference is formatting — shorter paragraphs, more sub-heads, and a buy button instead of a reply card." — Dan Kennedy

## The Framework: Web Sales Page Architecture

### Long-Form Sales Page Structure
```
1. PRE-HEADLINE (qualifier)
2. HEADLINE (main promise)
3. SUBHEADLINE (expand/support)
4. OPENING HOOK (2-3 sentences)
5. PROBLEM SECTION (agitate)
6. STORY (Epiphany Bridge)
7. SOLUTION INTRODUCTION
8. BENEFITS (fascination bullets)
9. PROOF (testimonials, data)
10. OFFER STACK (with values)
11. GUARANTEE
12. CTA BUTTON
13. URGENCY/SCARCITY
14. FAQ (objection handling)
15. FINAL CTA
16. P.S. SECTION
```

### Web-Specific Adaptations
- Shorter paragraphs (2-3 sentences max)
- Sub-headlines every 200-300 words
- Bold/highlight key phrases
- Images support the copy (don't replace it)
- Multiple CTA buttons throughout
- Mobile-responsive design

## Step-by-Step Process

### Step 1: Write Using Sales Letter Framework
Follow the 16-section structure above.

### Step 2: Adapt for Web Consumption
Shorter paragraphs, more visual breaks, scannable format.

### Step 3: Place Multiple CTAs
CTA buttons after proof, after offer, after guarantee, and at end.

### Step 4: Optimize for Mobile
60%+ of traffic is mobile — test readability on phone screen.

### Step 5: Add Trust Elements
SSL badge, payment logos, privacy statement, contact info.
